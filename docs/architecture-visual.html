Choosing a Policy Base Class Policy

SimplePolicy

Override simple_on_request, simple_on_response_content, simple_on_response_tool_call

Use when: you only need to inspect or transform finished text/tool calls. Content is buffered until block completion. Works with both OpenAI and Anthropic.

BasePolicy + OpenAIPolicyInterface + AnthropicExecutionInterface

Full streaming control. Custom block buffering. Multiple backend calls. Event reconstruction.

Use when: you need to control streaming behavior, buffer blocks yourself, make multi-turn backend calls, or implement complex patterns like overseer/judge policies. Per-request state via context.get_request_state(self, StateType, factory).

Policy Rules Important

  • Policies are singletons — never store request-scoped data on self. Use context.get_request_state().
  • Config-time collections must be immutable (tuple, frozenset). freeze_configured_state() enforces this at load time.
  • For SimplePolicy, streaming content is buffered until block completion, then your hook runs on complete content.
  • For full OpenAIPolicyInterface, you control what gets pushed to the egress queue via ctx.push_chunk().
