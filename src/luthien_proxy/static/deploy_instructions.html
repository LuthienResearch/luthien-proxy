$ Deploy Instructions

Get Claude Code running through Luthien Proxy in five steps. Copy-paste each command block to go from zero to policy-enforced AI coding.

Prerequisites
Docker
Python 3.13+
uv
Claude Code CLI
1

Clone the repository

Grab the source and move into the project directory.

bash 
git clone https://github.com/LuthienResearch/luthien-proxy
cd luthien-proxy
Don't have uv installed?

Install uv (a fast Python package manager):

bash 
curl -LsSf https://astral.sh/uv/install.sh | sh
2

Configure environment variables

Copy the example env file and add your real API keys.

bash 
cp .env.example .env

Open .env in your editor and set the following keys:

Variable Description
OPENAI_API_KEY Your OpenAI API key (e.g. sk-proj-...). Required if proxying OpenAI models.
ANTHROPIC_API_KEY Your Anthropic API key (e.g. sk-ant-...). Required for Claude models.
PROXY_API_KEY The key clients use to authenticate to the proxy. Default: sk-luthien-dev-key
ADMIN_API_KEY Key for the admin UI and management API. Default: admin-dev-key
Tip: For production, change PROXY_API_KEY and ADMIN_API_KEY to strong, unique values.
3

Start the proxy

The quick-start script validates dependencies, starts Docker services (PostgreSQL, Redis, gateway), runs migrations, and auto-selects free ports.

bash 
./scripts/quick_start.sh

When it finishes, note the gateway URL printed in the output (usually http://localhost:8000). The port may differ if 8000 was taken.

Verify the gateway is healthy
bash 
curl http://localhost:8000/health

You should see a JSON response with "status": "ok".

4

Launch Claude Code through the proxy

The easiest way is the built-in launch script, which sets up all environment variables automatically:

bash 
./scripts/launch_claude_code.sh
Manual setup (without the launch script)

Set these environment variables before running claude:

bash 
# Point Claude Code at the proxy (don't include /v1 — the SDK adds it)
ANTHROPIC_BASE_URL="http://localhost:8000/"

# Authenticate with your proxy API key
ANTHROPIC_AUTH_TOKEN="sk-luthien-dev-key"

# Launch Claude Code
env \
  ANTHROPIC_BASE_URL="http://localhost:8000/" \
  ANTHROPIC_AUTH_TOKEN="sk-luthien-dev-key" \
  claude
Important: Do not include /v1 in ANTHROPIC_BASE_URL. The Anthropic SDK appends it automatically.
Don't have Claude Code CLI installed?
bash 
npm install -g @anthropic-ai/claude-cli
5

Verify it works

Send a test request through the proxy to confirm everything is connected:

bash 
curl -s http://localhost:8000/v1/chat/completions \
  -H "Authorization: Bearer sk-luthien-dev-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o-mini",
    "messages": [{"role": "user", "content": "Say hello"}],
    "max_tokens": 50
  }'

You should receive a chat completion response. Open the Activity Monitor to see the request flow through in real-time.

Next steps: Configure policies via the Policy Config UI, or browse past sessions in Conversation History.

Troubleshooting

Gateway won't start / port conflict

The quick-start script auto-selects free ports. If you need a fixed port, uncomment GATEWAY_PORT=8000 in your .env file. Check for conflicts with:

bash 
lsof -i :8000
Docker containers failing

Check container status and logs:

bash 
docker compose ps
docker compose logs gateway --tail 50

If containers are stuck, try a clean restart:

bash 
docker compose down
./scripts/quick_start.sh
Claude Code can't connect to the proxy

Verify the gateway is healthy:

bash 
curl http://localhost:8000/health

Common issues:

• ANTHROPIC_BASE_URL should NOT include /v1 — the SDK appends it.

• Make sure you're using ANTHROPIC_AUTH_TOKEN (not ANTHROPIC_API_KEY) — the proxy needs the proxy key, not your real Anthropic key.

• The trailing slash in the URL matters: use http://localhost:8000/

Authentication errors (401 / 403)

The proxy uses two separate keys:

• PROXY_API_KEY — for LLM requests (used by Claude Code as ANTHROPIC_AUTH_TOKEN)

• ADMIN_API_KEY — for the admin UI and debug endpoints

Make sure the key in your client matches PROXY_API_KEY in your .env.
