Configure policies

bash 
ANTHROPIC_BASE_URL={{BASE_URL}} claude

Append any claude arguments as usual:

examples 
# interactive session
ANTHROPIC_BASE_URL={{BASE_URL}} claude

# run a one-shot prompt
ANTHROPIC_BASE_URL={{BASE_URL}} claude -p "explain this repo"

# continue the last conversation
ANTHROPIC_BASE_URL={{BASE_URL}} claude --continue

Your existing Anthropic credentials (API key or OAuth) pass through automatically. Do not append /v1 to the URL.

Claude Code can't connect

Check that ANTHROPIC_BASE_URL does not include /v1 — the SDK appends it automatically.

Verify the proxy is reachable: curl {{BASE_URL}}/health

If connecting from a different machine, use the host IP (not localhost).

Installed via native binary? Use npm instead

The native Claude Code binary (installed via curl | bash) bundles the Bun runtime, whose fetch() ignores HTTP_PROXY/HTTPS_PROXY environment variables. API streaming requests may bypass the proxy entirely.

Fix: install via npm instead: npm install -g @anthropic-ai/claude-code

ANTHROPIC_BASE_URL (direct URL override) is unaffected — this only matters if you also use a forward proxy.

Authentication: API key vs OAuth

Use ANTHROPIC_API_KEY (not ANTHROPIC_AUTH_TOKEN) for the most reliable behavior. Claude Code sends it as the x-api-key header, which the gateway expects.

If you use OAuth login (claude.ai account), the OAuth handshake hits Anthropic directly and does not go through the proxy. Use an API key to ensure all traffic routes through the gateway.
